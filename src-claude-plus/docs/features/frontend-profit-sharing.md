# 前端分账功能说明

## 1. 功能定位

前端分账功能用于在订单支付后，按照业务规则将订单金额拆分到不同参与方，并在前端提供配置、发起和查询能力。

## 2. 典型场景

- 平台与商户分账
- 平台与推广员分账
- 多参与方按比例分账

## 3. 前端能力范围

- 分账规则配置（比例、固定金额、优先级）
- 分账发起入口（按订单或批量）
- 分账结果查询（处理中、成功、失败）
- 分账明细查看（参与方、金额、时间、通道回执）

## 4. 建议页面结构

- 分账规则管理
- 分账任务列表
- 分账任务详情
- 分账异常重试

## 5. 权限建议

建议将以下权限点拆分为独立菜单或按钮权限：

- `pay:profit-sharing:rule:list`
- `pay:profit-sharing:rule:update`
- `pay:profit-sharing:task:create`
- `pay:profit-sharing:task:query`
- `pay:profit-sharing:task:retry`

## 6. 接口与字段说明（建议）

- 规则配置：规则名称、适用业务、参与方标识、分账方式（比例/固定）、生效状态
- 分账任务：订单号、任务号、总金额、分账金额、渠道状态、失败原因

请以实际后端 API 定义为准，前端字段命名建议保持与后端 VO 一致。

## 7. 风险与注意事项

- 金额计算需避免浮点误差，前端仅展示，最终金额以后端计算为准
- 重试操作必须有幂等约束（同一任务避免重复分账）
- 失败原因需可视化，便于运营排查
