# DevOps 构建脚本说明

本目录包含项目的构建和部署相关脚本，主要用于管理不同环境的配置文件和自动化构建流程。

## 文件结构

```
devops/
├── build.sh        # 主构建脚本
├── update-ip.sh    # IP地址更新脚本
└── README.md       # 本说明文件
```

## build.sh - 环境配置构建脚本

### 功能概述

`build.sh` 是项目的主要构建脚本，用于根据指定的环境参数自动选择和应用相应的环境配置文件。该脚本支持三种环境：本地开发环境（local）、测试环境（test）和生产环境（prod）。

### 主要功能

1. **环境配置管理**：根据传入的环境参数选择对应的配置文件
2. **配置文件替换**：将选定的环境配置文件复制为 `.env` 文件
3. **本地开发优化**：当使用本地环境时，自动执行 IP 地址更新脚本
4. **错误处理**：提供详细的错误信息和参数验证

### 使用方法

```bash
# 基本语法
./devops/build.sh <环境参数>

# 或者从项目根目录执行
bash devops/build.sh <环境参数>
```

### 支持的环境参数

| 参数 | 说明 | 对应配置文件 | 用途 |
|------|------|-------------|------|
| `local` | 本地开发环境 | `.e.local` | 本地开发调试，会自动更新IP地址 |
| `test` | 测试环境 | `.e.test` | 测试环境部署 |
| `prod` | 生产环境 | `.e.prod` | 生产环境部署 |

### 使用示例

#### 1. 配置本地开发环境
```bash
./devops/build.sh local
```
执行结果：
- 将 `.e.local` 复制为 `.env`
- 自动执行 `update-ip.sh` 更新本机IP地址
- 更新 `SHOPRO_DEV_BASE_URL` 和 `SHOPRO_BASE_URL` 为本机IP

#### 2. 配置测试环境
```bash
./devops/build.sh test
```
执行结果：
- 将 `.e.test` 复制为 `.env`
- 使用测试环境的配置参数

#### 3. 配置生产环境
```bash
./devops/build.sh prod
```
执行结果：
- 将 `.e.prod` 复制为 `.env`
- 使用生产环境的配置参数

### 脚本执行流程

1. **参数验证**：检查是否提供了正确的环境参数
2. **路径定位**：自动定位项目根目录
3. **配置文件检查**：验证对应的环境配置文件是否存在
4. **配置文件复制**：将环境配置文件复制为 `.env`
5. **本地环境特殊处理**：如果是本地环境，执行IP更新脚本
6. **完成确认**：输出操作完成信息

### 环境配置文件说明

项目根目录下包含以下环境配置文件：

- **`.e.local`**：本地开发环境配置
  - 包含本地开发所需的API地址、调试参数等
  - 会被 `update-ip.sh` 脚本动态更新IP地址
  
- **`.e.test`**：测试环境配置
  - 包含测试服务器的API地址和相关配置
  - 用于测试环境的部署和验证
  
- **`.e.prod`**：生产环境配置
  - 包含生产服务器的API地址和优化配置
  - 用于正式环境的部署

- **`.env`**：当前激活的环境配置
  - 由构建脚本自动生成
  - 应用程序实际读取的配置文件

## update-ip.sh - IP地址更新脚本

### 功能概述

`update-ip.sh` 是专门用于本地开发环境的辅助脚本，能够智能获取本机IP地址并自动更新环境配置文件中的相关URL配置。

### 主要功能

1. **智能IP获取**：自动检测并获取本机可用的IP地址
2. **网卡优先级**：按优先级顺序检查网卡（en0、eth0、wlan0等）
3. **IP地址过滤**：过滤掉无效的IP地址（如127.0.0.1、169.254.x.x等）
4. **配置更新**：自动更新 `SHOPRO_DEV_BASE_URL` 和 `SHOPRO_BASE_URL`

### IP获取策略

1. **优先网卡检查**：按顺序检查 en0、eth0、wlan0、en1、eth1
2. **网段优先级**：优先选择 192.168.1.x 网段的IP
3. **私有网络优先**：优先选择私有网络地址（192.168.x.x、10.x.x.x、172.16-31.x.x）
4. **公网IP备选**：如果没有私有网络地址，使用公网IP

### 自动更新的配置项

- `SHOPRO_DEV_BASE_URL`：开发环境基础URL
- `SHOPRO_BASE_URL`：基础URL
- 固定端口：48080

## 错误处理

### 常见错误及解决方案

1. **"错误：请提供环境参数"**
   - 原因：未提供环境参数
   - 解决：执行时添加 local、test 或 prod 参数

2. **"错误：无效的环境参数"**
   - 原因：提供了不支持的环境参数
   - 解决：确保参数为 local、test 或 prod 之一

3. **"错误：未找到环境文件"**
   - 原因：对应的环境配置文件不存在
   - 解决：检查项目根目录下是否存在相应的 .e.* 文件

4. **"错误：未能获取任何有效的IP地址"**
   - 原因：update-ip.sh 无法获取本机IP
   - 解决：检查网络连接，确保网卡正常工作

## 最佳实践

### 开发流程建议

1. **开始开发前**：
   ```bash
   ./devops/build.sh local
   npm run dev
   ```

2. **测试环境部署**：
   ```bash
   ./devops/build.sh test
   npm run build
   ```

3. **生产环境部署**：
   ```bash
   ./devops/build.sh prod
   npm run build
   ```

### 注意事项

1. **文件权限**：确保脚本具有执行权限
   ```bash
   chmod +x devops/build.sh
   chmod +x devops/update-ip.sh
   ```

2. **环境文件管理**：
   - 不要直接编辑 `.env` 文件，它会被构建脚本覆盖
   - 环境特定的配置应该在对应的 `.e.*` 文件中修改

3. **版本控制**：
   - `.env` 文件通常应该被 `.gitignore` 忽略
   - `.e.*` 文件应该被版本控制管理

4. **本地开发**：
   - 本地开发时，IP地址会自动更新，无需手动配置
   - 如果IP地址变化，重新运行 `./devops/build.sh local` 即可
